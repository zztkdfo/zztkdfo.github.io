### 엔티티 설정
- - -
```
@Getter
@Setter
@Entity
@Table(name ="TB_MEMBER",
    uniqueConstraints={
        @UniqueConstraint(
            name="INDEX_MEMBER_CODE",
            columnNames = {"MEMBER_CODE"}
        )
    }
)
public class Member {

    @Id
    @Column(name = "MEMBER_ID")
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "MEMBER_CODE", nullable = false)
    private String memberCode;

    @Column(name = "MEMBER_NAME")
    private String memberName;

    @Column(name = "MEMBER_AGE")
    private Integer memberAge;

    @JsonIgnore
    @ManyToOne
    @JoinColumn(name = "TEAM_ID")
    private Team team;

    @Column(name = "DESCRIPTION", columnDefinition="varchar(400) default 'EMPTY'")
    private String description;


}
```
#### @Getter, @Setter
Getter, Setter 자동 생성 Lombok 라이브러리
#### @Entity
JPA를 사용해서 테이블과 매핑할 클래스에 @Entity 어노테이션을 필수로 붙여 사용합니다.
#### @Table
엔티티와 매핑할 테이블을 지정합니다. @Table을 설정하지 않으면 엔티티명(Member)이름을 테이블 명으로 사용합니다. 따라서 Table명을 설정 하고 싶을 경우, 필자와 같이 name 속성을 사용하시길 바랍니다. uniqueConstraints 속성은 DDL 생성 시에 유니크 제약조건을 만들 수 있습니다.
#### @Id
@Id 어노테이션을 사용해서 기본 키를 애플리케이션에서 직접 할당 할 수 있습니다.
JPA가 제공하는 데이터베이스 기본 키 생성 전략은 직접 할당, 자동생성(IDENTITY, SEQUENCE, TABLE)전략이 있습니다. @GeneratedValue 어노테이션을 사용해서 만들 수 있습니다.
#### @Column
@Column 어노테이션은 객체 필드를 테이블 컬럼에 매핑하는 기능입니다. 가장 많이 사용되는 어노테이션이며 다양한 기능을 제공합니다.
| 속성 | 기능     | etc     |
| :------------- | :------------- |:------------- |
| name       | 필드와 매핑할 테이블의 컬럼명       ||
| insertable       | 해당 컬럼에 대한 insert 여부, false 옵션을 경우 읽기 전용일 때 사용       |기본값 : true|
| updatable       | 해당 컬럼에 대한 update 여부, false 옵션을 경우 읽기 전용일 때 사용       |기본값 : true|
| nullable       | null 허용 여부       |기본값 : true|
| columnDefinition       | 데이터베이스 컬럼 정보를 직접 설정       |       |
| length       | 문자 길이       |  |
